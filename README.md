# 🌐 Jakarta EE - Fundamentos esenciales

Este repositorio reúne los **fundamentos esenciales de Jakarta EE** que todo desarrollador Java debería conocer.  
El objetivo es construir una base sólida sobre los estándares de Jakarta antes de dar el salto a frameworks modernos
como Quarkus.

---

## 📌 ¿Qué es Jakarta EE?

`Jakarta EE (Enterprise Edition)` es el conjunto de especificaciones abiertas y estandarizadas para el desarrollo de
aplicaciones empresariales en Java. Es el sucesor de `Java EE` y define APIs que permiten construir aplicaciones
portables, escalables y mantenibles.

### ✨ Características principales

- 📡 **Jakarta RESTful Web Services (JAX-RS)** → creación de APIs REST con anotaciones estándar.
- 🧩 **Jakarta CDI (Contexts and Dependency Injection)** → sistema de inyección de dependencias que da vida a los
  componentes.
- 💾 **Jakarta Persistence (JPA)** → manejo de entidades y persistencia en bases de datos relacionales.
- ✅ **Jakarta Validation (Bean Validation)** → validación declarativa de datos con anotaciones.
- 🔒 **Jakarta Transactions (JTA)** → control de transacciones para garantizar consistencia en operaciones.

## 🎯 Objetivo del repositorio

- Reunir ejemplos prácticos y proyectos pequeños para cada módulo esencial.
- Documentar conceptos clave con explicaciones claras y visuales.
- Servir como referencia neutral de Jakarta EE, pero con la mirada puesta en frameworks modernos.
